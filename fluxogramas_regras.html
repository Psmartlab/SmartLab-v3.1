9. Dados em Tempo Real (Firestore)

AccessControlContext — Fontes de dados e sincronização

Fluxo de Carregamento ACL

flowchart TD A["App inicia com usuário autenticado"] --> B["AccessControlProvider monta"] B --> C["Inicia 2 subscriptions simultâneas"] C --> D["onSnapshot: settings/rolePermissions"] C --> E["onSnapshot: collection rules/"] D --> F["Atualiza state: rolePermissions"] E --> G["Atualiza state: screenRules"] F --> H{"Ambos carregados?"} G --> H H -- Não --> I["aclLoading = true"] H -- Sim --> J["aclLoading = false"] J --> K["useAccessControl disponível para uso"] subgraph "Firestore Collections" FS1["📄 settings/rolePermissions
Mapa: permKey → Role → bool"] FS2["📁 rules/ (collection)
Cada doc = 1 regra dinâmica"] end D -.-> FS1 E -.-> FS2 style K fill:#dcfce7,stroke:#16a34a,color:#14532d
Real-time: Qualquer edição nas permissões ou regras do Firestore reflete instantaneamente em todos os clientes conectados, sem necessidade de refresh.
Cleanup: As subscriptions são removidas quando o componente desmonta ou o user.uid muda.

Estrutura de uma Regra (Firestore doc)

flowchart LR R["Documento Rule"] --> ID["id: string (auto)"] R --> NM["name: 'Bloquear User em Settings'"] R --> DS["description: 'Texto livre'"] R --> AC["active: true/false"] R --> PR["priority: 10 (menor = maior prior.)"] R --> CT["conditionType: AND / OR"] R --> CO["conditions: Condition[]"] R --> AT["action: type + target"] CO --> C1["field: 'user.role'
operator: '=='
value: 'User'"] CO --> C2["field: 'screen'
operator: '=='
value: 'screen:settings'"] AT --> AT1["type: 'deny'"] AT --> AT2["target: 'screen'"]
